01 - THE PROBLEM

You have a drum loop recorded at 120 BPM. Your current project is at 135 BPM. When you load that loop onto a track, press [PLAY] and it plays too fast AND at the wrong pitch. Without time-stretching, the loop is essentially useless in your project.

THE SOLUTION

Timestretch (TSTR) adapts the loop to match your project tempo WITHOUT changing its pitch. The drum kick stays fat, the vocal stays natural, but everything plays at the right speed.

This is one of the most powerful features of the Octatrack because it lets you:

  • ✓ Mix loops at different tempos into one project
  • ✓ Change your project tempo mid-jam using [FUNC] + [TEMPO] without loops sounding wrong
  • ✓ Keep long static samples (no RAM cost) while keeping them tempo-locked
  • ✓ Work with material from Ableton, loops packs, anywhere

02 - HOW TIMESTRETCH WORKS ON THE OT

Timestretch requires THREE things to be configured. All three must be set for it to work properly.

THE THREE-STEP SETUP

Step 1: ORIGINAL TEMPO — Tell the OT the loop's TRUE BPM in the [AUD] Audio Editor

Step 2: TIMESTRETCH MODE — Choose the algorithm (NORMAL for tonal, BEAT for drums)

Step 3: TSTR = AUTO — Enable timestretch in [PLAYBACK] Setup

Step-by-Step Navigation

1

OPEN AUDIO EDITOR

While viewing your sample on any [TRACK], press:

[FUNC] + [AUD]
2

NAVIGATE TO ATTRIBUTES

Press the [RIGHT] arrow key to browse pages of the [AUD] Audio Editor. You're looking for "Attributes" (usually page 2 or 3).

3

SET ORIGINAL TEMPO

Use the [LEVEL] encoder to adjust. If your loop is 120 BPM, set this to 120. This tells the OT: "This sample was recorded at 120 BPM."

4

SET TIMESTRETCH MODE

Use the [RIGHT] arrow to move to the TIMESTRETCH field. Choose:

  • NORMAL — for tonal material (pads, vocals, melodic)
  • BEAT — for drums, percussion, rhythmic loops
5

EXIT [AUD] AUDIO EDITOR

Press [NO] to return to the main sequencer view.

6

OPEN [PLAYBACK] SETUP

Select the [TRACK], then press:

[FUNC] + [PLAYBACK]
7

SET TSTR = AUTO

Find the TSTR field and use the [LEVEL] encoder to set it to AUTO. This enables the timestretch effect globally for this sample.

8

TEST IT!

Press [PLAY] to start the sequencer. Change your project tempo using [FUNC] + [TEMPO]. The loop should now follow your tempo changes without sounding wrong.

03 - BPM AUTO-DETECTION

The Octatrack has a clever feature: it reads numbers from filenames and assumes they're BPM values.

IMPORTANT RANGE

The OT only auto-detects numbers in the range 85–170 BPM . Numbers outside this range are ignored.

How It Works

Filename OT Detects? Result
DrumLoop_128bpm.wav YES ✓ Sets ORIGINAL TEMPO to 128 automatically
Bass_90.wav YES ✓ Sets ORIGINAL TEMPO to 90 automatically
Sample_140_Dark.wav YES ✓ Sets ORIGINAL TEMPO to 140 (even if "140" is unrelated!)
Recording_2024.wav NO ✗ 2024 is outside range. You must set ORIGINAL TEMPO manually.
Pad_96_Ambient.wav YES ✓ Sets ORIGINAL TEMPO to 96 automatically

⚠️ THE GOTCHA

If your filename is "Sample_140_Dark.wav" but the loop is actually 120 BPM, the OT will set ORIGINAL TEMPO to 140 and timestretch will sound wrong. Always manually verify ORIGINAL TEMPO in the [AUD] Audio Editor (press [FUNC] + [AUD] to open), especially with ambiguous filenames.

Naming Convention (Best Practice)

For easy BPM auto-detection, use this naming scheme:

Type_Description_BPMvalue.wav Examples: Drum_DarkKick_128bpm.wav Bass_Sub_95bpm.wav Pad_Ambient_85bpm.wav Loop_Vocal_Chop_140bpm.wav

04 - NORMAL VS BEAT ALGORITHM

The Octatrack's timestretch engine has two modes, selectable in the [AUD] Audio Editor attributes page. They sound and behave very differently.

Property NORMAL BEAT
Best For Tonal material: pads, melodic loops, vocals, strings, keys Drums, percussion, rhythmic loops, breaks
How It Works Granular time-stretching (stretches the waveform itself) Slice-based (detects transients, repositions them)
Artifacts at Extremes Slight granular shimmer or watering effect May stutter or have pumping feel
Quality at Extreme Stretch Good for moderate stretching (±20%) Good for rhythmic material (even ±50%)

Quick Decision Tree

CHOOSE YOUR MODE

  • Is it a drum loop ? → Use BEAT
  • Is it a bass loop ? → Use NORMAL
  • Is it a vocal chop ? → Use NORMAL
  • Is it a break (drums)? → Use BEAT
  • Is it a pad or string ? → Use NORMAL
  • Is it mixed material (drums + bass)? → Try BEAT first, then experiment

Experiment! Both modes sound different on the same material. If BEAT sounds bad on a drum loop, try NORMAL. Open the [AUD] Audio Editor with [FUNC] + [AUD], navigate with [RIGHT] to attributes, and switch modes. The OT's CPU can handle it either way — it's about which sound you prefer.

05 - STATIC AND FLEX BOTH SUPPORT TSTR

This is a common misconception: "Timestretch only works on Static machines, not Flex." Check the [SRC] page to see which machine type is assigned to your [TRACK].

THE TRUTH

TSTR works on BOTH Static and Flex machines. You can use timestretch regardless of which machine type you choose.

When to Use Which

Scenario Machine Why
Long loop (30+ sec) that needs tempo sync, no slicing needed Static + TSTR Zero RAM cost. Loop streams from card and stretches to tempo. Perfect for backing tracks.
Long loop (30+ sec) that you want to slice and chop Flex + TSTR Costs RAM, but you can slice it into 64 pieces and trigger slices p-locked. Slices also timestretch.
Short loop (1-5 sec) tempo-matched Flex + TSTR Low RAM cost (~0.5 MB per loop). Instant access for slicing and manipulation.
Resampled recording from recorder track Flex + TSTR Recordings are short. TSTR will match them to your current tempo.

Static Machines are Underrated for Long Loops

If you have a 2-minute drum loop and you just want to play it straight (no slicing), use:

STATIC MACHINE SETUP

  1. Assign Static to a [TRACK] (press [TRACK] to select, then assign via [SRC] setup)
  2. Load your loop via the sample browser (hold a [TRIG 1] key + [UP])
  3. Set ORIGINAL TEMPO + TIMESTRETCH in [AUD] Audio Editor ([FUNC] + [AUD])
  4. Set TSTR = AUTO in [PLAYBACK] Setup ([FUNC] + [PLAYBACK])
  5. ZERO RAM used. Loop streams from CF card. Press [PLAY] — tempo-matched.

This is the most efficient way to use long samples in the OT.

06 - PRACTICAL EXERCISE: FIX A LOOP

Let's walk through the complete workflow. You'll need any sample or loop on your Octatrack or CF card.

SETUP

Choose a drum loop or any audio sample you have. We'll set it up so it follows your project tempo.

1

LOAD A SAMPLE ONTO A TRACK

Select any [TRACK] using the [T1] through [T8] track keys. Open the sample browser (hold [TRIG 1] + [UP]). Pick any sample from your AUDIO/ folder and press [YES] to load it.

2

OPEN AUDIO EDITOR

Press:

[FUNC] + [AUD]
3

FIND ATTRIBUTES PAGE

Press the [RIGHT] arrow key to navigate pages. Use [LEFT] and [RIGHT] to browse. Stop when you see "ORIGINAL TEMPO" and "TIMESTRETCH" fields.

4

SET ORIGINAL TEMPO

Use the [LEVEL] encoder to set ORIGINAL TEMPO to the sample's true BPM. If unsure, check the filename or try 120 BPM as a baseline.

5

SET TIMESTRETCH MODE

Use the [LEVEL] encoder to move to TIMESTRETCH. If it's drums, choose BEAT. If it's tonal, choose NORMAL. If mixed, try BEAT first.

6

EXIT [AUD] AUDIO EDITOR

Press [NO] to exit back to the main view.

7

OPEN [PLAYBACK] SETUP

Select your [TRACK] (if not already selected). Press:

[FUNC] + [PLAYBACK]
8

ENABLE TSTR = AUTO

Find the TSTR field and use the [LEVEL] encoder to set it to AUTO. This globally enables timestretch for this sample.

9

TEST TEMPO CHANGES

Press [YES] to confirm. Place a trig on step [TRIG 1] and press [PLAY] to hear the sample. Now use [FUNC] + [TEMPO] to change your project BPM. Press [STOP] then [PLAY] again. The sample should follow your tempo without sounding wrong!

✓ WHAT YOU'VE LEARNED

You've configured a sample to be tempo-aware. This sample will now automatically match any project tempo you set, so you can change your groove without loops sounding wrong.

07 - EDGE CASES & GOTCHAS

Half-Time and Double-Time Loops

Sometimes a loop sounds like it's playing at double the speed. This usually means the sample was recorded at a different effective tempo.

HALF-TIME LOOP EXAMPLE

You load a loop and press [PLAY] but it sounds like it's playing at 2x speed. The filename says 140 BPM, but it sounds wrong. Open the [AUD] Audio Editor ([FUNC] + [AUD]) and try setting ORIGINAL TEMPO to 70 (half of 140). Often samples are recorded at double-time internally.

DOUBLE-TIME LOOP EXAMPLE

Conversely, a loop sounds too slow. Open [FUNC] + [AUD] and try doubling the BPM value in ORIGINAL TEMPO using the [LEVEL] encoder.

Extreme Stretching (>50% Change)

If you stretch a loop more than 50% faster or slower, you'll hear artifacts:

  • NORMAL mode: Granular shimmer, watering effect
  • BEAT mode: Stuttering, pumping feel

This is normal. Press [STOP], adjust the [TEMPO] with [FUNC] + [TEMPO], and if it sounds bad, consider using samples closer to your project tempo.

Resampled Material Does NOT Auto-Follow Tempo

If you resample a pattern (press [REC] to arm the recorder on a [TRACK] with a Flex machine) and then try to timestretch the resampled audio, it won't follow tempo changes. Recordings are "locked-in" to the tempo they were recorded at.

If you need a resampled loop to be tempo-aware, resample it again at your new tempo.

The Ableton → Octatrack Pipeline

If you're exporting loops from Ableton:

  1. Warp the loop to your target tempo in Ableton
  2. Consolidate or export at exact bar lengths (1/2/4/8 bars)
  3. Export as WAV: 44.1 kHz, 16 or 24-bit
  4. Include BPM in filename: e.g., "Drums_Chop_135bpm.wav"
  5. Drag into AUDIO/ folder on your OT card
  6. Load on OT and the BPM is auto-detected!
