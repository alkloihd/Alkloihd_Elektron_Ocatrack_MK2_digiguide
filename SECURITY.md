# Security Policy

## Scope

This public repository is a landing/status surface and does not host production secrets or deployable runtime code.

## Reporting a Security Issue

If you discover a security issue in any related deployed product:

- Open a GitHub issue with prefix `[SECURITY]` and avoid posting exploit details publicly.
- For sensitive reports, use direct creator contact channels listed in `README.md`.

## Handling Policy

- Reports are acknowledged as quickly as possible.
- Verified issues are prioritized by severity and user impact.
- Public disclosure should wait until remediation is complete.
