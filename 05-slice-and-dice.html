SLICE & DICE
⏱ 30 MIN
⚡ BEGINNER/INT

1. WHAT ARE SLICES?

Slices are how you divide a sample into rhythmic chunks. Think of it like cutting a drum loop into individual hits. Each slice becomes its own playable segment, numbered 1-64 (max 64 slices per sample). You create and manage slices in the Audio Editor, opened with [FUNC] + [AUD].

Once sliced, you can play any slice independently using the [TRIG] keys, chop them into a new rhythm, reverse them, pitch them, or sequence them with [REC] (Grid Recording). This is the foundation of creative loop chopping on the Octatrack.

KEY CONCEPT
A drum loop at 128 BPM might be 8 bars long. If you grid-slice it into 32 slices, you get roughly 2 slices per beat (32 slices ÷ 16 steps = 2 per beat). Now you can play those slices in any order, creating completely new breakbeats.

2. THE AUDIO EDITOR

The Audio Editor is where you view, slice, and edit samples. To access it:

MENU PATH
1. Select a track with a sample loaded
2. Press [FUNC] + [AUD] to open Audio Editor
3. You'll see the waveform displayed

Navigation in Audio Editor:

  • [LEFT]/[RIGHT] arrows - move playhead through sample
  • [FUNC] + [LEFT]/[RIGHT] - zoom in/out on waveform
  • [UP]/[DOWN] - adjust waveform zoom depth
  • [YES]/[NO] - confirm/cancel slicing actions

The waveform shows silence as flat lines and loud sections as tall peaks. You'll use these visual cues to find where to place slice markers.

3. GRID SLICING: THE INTERACTIVE WAY

Grid slicing divides a sample into EVENLY SPACED sections. Perfect for drum loops that are locked to a tempo. Try it below:

Select number of slices and click APPLY GRID. Each vertical line = one slice.
MENU PATH
Audio Editor > [FUNC] (to access menu) > Slice > CREATE SLICE GRID
Select number of divisions (4, 8, 16, 32, 64) > [YES] to confirm

When to use grid slicing:

  • OK Drum loops locked to tempo
  • OK 4/4 time signatures (most loops)
  • OK When you want even rhythmic divisions
  • OK Quick way to slice without thinking

4. MANUAL SLICING: PRECISION CHOPPING

For irregular samples or precise control, manual slicing lets you place markers exactly where you want. Here's the process:

STEP-BY-STEP
1. In Audio Editor, use [LEFT]/[RIGHT] to find your slice point
2. Press [YES] to add a marker at that position
3. The marker appears as a vertical line in the waveform
4. Continue placing markers throughout the sample
5. Press [NO] when done, then confirm to save slices

CRITICAL: Zero-Crossing Snap

Always enable ZERO-CROSSING SNAP before slicing. This automatically places markers at points where the waveform crosses zero (silence). Why? Prevents clicks and pops at slice boundaries when you play them back.

ZERO-CROSSING SNAP
Audio Editor menu > Snap > ZERO CROSSING
Enable this, then place markers. The OT will snap them to the nearest zero crossing automatically.

Manual slicing takes longer than grid slicing, but gives you surgical precision. Perfect for one-shots, vocal chops, or complex loops where grid divisions don't make sense.

5. PLAYING SLICES: TURNING CHUNKS INTO BEATS

Once your sample is sliced, you need to enable slice playback in the track's Playback Setup:

ENABLE SLICE PLAYBACK
[TRACK] button on the sliced track > Playback Setup > SLICE = ON

Now each [TRIG] key can play a specific slice. Here's the interactive demo below - you're building a chopped breakbeat from 16 slices. Click each step to assign a slice number:

SLICE SEQUENCER (16 STEPS)
HOW TO P-LOCK SLICES
1. In GRID RECORDING mode ([REC])
2. Hold [TRIG] key you want to lock
3. Turn the encoder to select slice number (1-64)
4. Release [TRIG]
5. That step now plays the selected slice

The sequencer above shows what a chopped beat looks like. Notice how steps are assigned to different slices? That's you rebuilding the rhythm from the original sample.

6. SLICE LOCK TECHNIQUES: BEYOND THE BASICS

Slices are just the start. You can manipulate each slice independently:

TECHNIQUE HOW WHY
Reverse slices Hold [TRIG] + turn RATE to negative (-100) on [PLAYBACK] page Backwards hits, glitch effects
Pitch slices Hold [TRIG] + turn TUNE on [PLAYBACK] page Tuned drum kit, melodic loops
Repeat slices Place same slice number on consecutive [TRIG] steps Stutter effects, rhythmic emphasis
Skip slices Leave [TRIG] steps empty in your sequence Syncopation, variation, space
Randomize with [LFO] [LFO] targeting SLICE parameter Generative patterns, chaos
EXAMPLE: GLITCH BEAT
1. Slice a drum loop into 32 pieces in the Audio Editor ([FUNC] + [AUD])
2. Enter [REC] mode and sequence slices 1-16 on [TRIG] keys
3. On [TRIG 9], hold it and reverse the slice (RATE = -100 on [PLAYBACK])
4. On [TRIG 13], hold it and pitch up 7 semitones (TUNE)
5. On [TRIG 15], repeat step 14's slice twice
Result: a recognizable loop with glitchy variation

7. .OT FILES & SAMPLE CHAINS (ADVANCED)

When you slice a sample on the OT, it creates a .ot metadata file alongside the WAV. This file stores:

  • Slice point locations (exact sample numbers)
  • Original tempo and key
  • Gain and loop settings
  • Playback mode
SAMPLE CHAINS
A chain = multiple sounds concatenated into ONE WAV file + ONE .ot file. Each "sound" becomes a slice. So 64 samples become 64 slices in a single slot = massive space savings.

Why use chains?

  • OK 64 drum hits in 1 slot instead of 64 slots
  • OK Saves RAM (one Flex allocation)
  • OK Faster navigation (fewer files)
  • OK Keep related samples together

Build chains with:

  • OctaChainer (free Mac app, UI-based)
  • Python + ffmpeg (CLI, if you're scripting)

Best practice: organize chains by power-of-2 divisions (16, 32, 64 slices). Easier to reason about slice boundaries.
