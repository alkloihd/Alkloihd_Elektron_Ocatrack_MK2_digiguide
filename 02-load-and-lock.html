    THE RAM BUDGET (85.5 MB Total)

    The Octatrack's memory is finite. Every Flex sample you load consumes RAM. Static machines consume ZERO RAM because they stream directly from the CF card. Understanding this trade-off is the key to smart sample loading.

    Total Available

    • 85.5 MB total RAM
    • 64 MB default allocated to Flex (adjustable in System settings)
    • Recorder buffers share Flex RAM

    RAM Costs Per Minute

    24-bit stereo: ~8 MB/min
    24-bit mono: ~4 MB/min
    16-bit stereo: ~5.3 MB/min
    16-bit mono: ~2.6 MB/min

    STATIC MACHINES: THE LAZY OPTION

    What They Do

    • Stream directly from CF card (no loading into RAM)
    • ZERO RAM cost, no matter how long the file
    • Can load files up to 2 GB in size
    • Perfect for backing tracks, stems, long ambient pads

    Strengths

    • No RAM budget concerns—load massive files freely
    • Great for long loops, full songs, backing tracks
    • Timestretch works on Static machines too
    • 128 Static slots per project (plenty of room)

    Gotchas

    • Heavy STRT (start position) modulation can cause streaming glitches
    • Not ideal for slicing or rapid position jumps
    • Can't edit slice points like you can with Flex

    Pro Tip

    Use Static machines for long loops (30+ seconds) and Flex for everything else. It's the fastest way to make smart decisions.

    FLEX MACHINES: THE POWER PLAYERS

    What They Do

    • Load entire audio sample into RAM for instant access
    • Perfect for slicing, chopping, retriggering
    • Costs RAM proportional to sample length
    • 128 Flex slots per project

    Strengths

    • Instant access to any position—jump around freely
    • Can create slices and chop patterns
    • Rapid retriggering (great for drums, hi-hats)
    • Timestretch works beautifully

    Gotchas

    • Limited by 64 MB Flex RAM (Recorder buffers share this space)
    • Every sample consumes RAM—budget carefully
    • If you run out of RAM, you can't load more Flex samples

    DECISION TREE: STATIC OR FLEX?

    Use this flowchart to decide which machine type to use:

    LOADING YOUR FIRST SAMPLE

    Step-by-step: how to load a sample onto a track. Here's the button sequence:

    1
    SELECT YOUR TRACK
    Press [TRACK] then [TRIG 1-8] to select which track to load the sample onto. The track LED will light up.
    2
    OPEN PLAYBACK SETUP
    Press [FUNC] then use the MAIN ENCODER to navigate to "PLAYBACK SETUP". Press [YES] to open.
    3
    CHOOSE MACHINE TYPE
    Navigate to "MACHINE" and select either STATIC or FLEX using the encoder. Use your decision tree knowledge here!
    4
    OPEN SAMPLE BROWSER
    Press [FUNC] then [CUE/SRC] to open the sample browser. You should see your AUDIO folder.
    5
    NAVIGATE AND SELECT
    Use the MAIN ENCODER to navigate through folders. Select your sample and press [YES] to load it.
    6
    TRIGGER IT
    Press any of the TRIG keys (1-16) to trigger the sample on that track. You should hear it play!

    MONITORING RAM USAGE

    Always check your RAM usage before loading new Flex samples. Here's where to look:

    1. Press [FUNC] to open the menu
    2. Navigate to PROJECT
    3. Select CONTROL
    4. Open MEMORY page

    This screen shows you how much Flex RAM is used and available. Keep an eye on this before loading new samples. If you're over 90% capacity, consider:

    • Using Static machines for new samples
    • Deleting unused Flex samples from RAM
    • Using sample chains to consolidate multiple sounds

    SAMPLE CHAINS: MAXIMIZE YOUR SLOTS

    What's a Chain?

    A sample chain is multiple audio samples concatenated into one WAV file, paired with a .ot metadata file that stores slice points. Instead of using 64 slots for 64 sounds, you can use 1 slot for 64 sounds.

    Why Use Chains?

    • Save slots: 64 sounds in 1 slot instead of 64 slots
    • Better for drum kits: one chain = whole kit
    • Works on Static machines (zero RAM cost)
    • You can p-lock through different hits in the chain per step

    Best Practices

    • Use power-of-2 slices: 16, 32, or 64 sounds per chain
    • Equal length samples work best (all kicks roughly same duration)
    • Tools: OctaChainer (free, macOS) or ffmpeg+Python

    How to Use on the OT

    1. Load the chain WAV file like any other sample
    2. In Audio Editor, slices are pre-defined by the .ot metadata
    3. In Playback Setup, enable SLICE = ON
    4. P-lock slice numbers per step to play different hits
    TEST YOURSELF
    Q1: Which machine uses ZERO RAM?
    Correct! Static machines stream from the card—zero RAM cost.
    Q2: A 24-bit stereo 2-minute sample costs how much RAM in Flex?
    Correct! 24-bit stereo costs 8 MB/min. 2 min = 16 MB.
    Q3: Can Static machines do timestretch?
    Correct! TSTR works on Static AND Flex machines.
    Q4: Max Flex slots per project?
    Correct! 128 Flex slots and 128 Static slots per project.
