    WELCOME TO THE OCTATRACK

    You're holding one of the most powerful and creative devices in electronic music production. The Elektron Octatrack MK2 is an 8-track hardware sampler and sequencer with real-time sampling, dual effects per track, a [CROSSFADER], and integration with external gear.

    This tutorial series will take you from first boot to confidently chopping, resampling, and performing live. We'll cover the fundamental concepts that make the OT tick, and show you why it's not just a sampler—it's an instrument.

    What You'll Learn Today

    • The OT's core architecture: Sets, projects, banks, patterns, parts, and scenes
    • How the save system works (.work vs .strd files)
    • The non-negotiable golden rules that protect your work
    • How your CF card is organized

    Learning Curve Heads-Up

    The OT has a steep learning curve, but it's intentional — every button and menu serves a purpose. We'll learn systematically, starting with the essentials. By the end of this series, you'll understand not just HOW to do things, but WHY the OT is designed this way. You'll know when to reach for [FUNC], when to press [YES], and how [PLAY] and [STOP] keep your workflow flowing.

    HARDWARE OVERVIEW

    The OT MK2 is organized into zones: trig keys, track buttons, the main [LEVEL] encoder, [CROSSFADER], and real-time controls. Here's the layout:

    KEY ZONES
    [TRIG 1]-[TRIG 8]: trigger samples / select patterns
    A/B IN: main stereo input pair
    [T1]-[T8]: select active track
    C/D IN: auxiliary input pair
    [FUNC]: hold for secondary functions
    MAIN OUT: stereo output to speakers
    [CROSSFADER]: morph between [SCENE A]/[SCENE B]
    [CUE] OUT: headphone preview output
    ENCODERS A-F: parameter control
    [LEVEL] ENCODER: master volume / navigate

    PROJECT HIERARCHY (CRITICAL)

    The OT has a very specific folder structure. Understanding this is essential — moving or renaming folders breaks everything. Use [FUNC] + [YES] to access the project menu, and press [BANK] to switch between the 16 banks (A-P).

    CF CARD (16 GB)
    The CF card is the Octatrack's external storage. FAT32 formatted with a 1024 file per-folder limit.
    └─ SET (top-level folder)
    CRITICAL: Renaming this breaks all projects because they reference ../AUDIO/ relative to SET. NEVER rename it.
    ├─ AUDIO/ (sample pool)
    All samples must live here. This is the shared pool across ALL projects. The OT can only find samples via ../AUDIO/. Never move samples—this breaks project references.
    ├─ Project Folders
    Each project folder contains: project.work (live state), project.strd (saved state), and A.strd through P.strd (16 banks).
    ├─ 16 Banks (A-P)
    Press [BANK] to select a bank. Each bank contains 16 patterns (selected with [TRIG 1]-[TRIG 8]) and 4 parts (switch with [PART]). Each part stores machine assignments, [FX1]/[FX2] settings, and 16 scenes.
    ├─ 4 Parts per Bank
    CRITICAL: Parts are PER BANK, not per pattern. Press [PART] to see which Part is active. If two patterns share a Part, editing one affects both. Use different Parts to isolate. Each Part holds [SRC], [AMP], [LFO], [FX1], and [FX2] settings for all 8 tracks.
    TL;DR: CF Card → SET → AUDIO (samples) + Project Folders. Each project → Banks (press [BANK]) → Parts (press [PART]) → Scenes (press [SCENE A] or [SCENE B]). Never rename SET or move samples. If you do, everything breaks.

    SAVE SYSTEM: .WORK VS .STRD

    The OT has a dual-state save system. Understanding it is crucial to protecting your work. To save manually, press [FUNC] + [YES] and select SAVE. To reload from your last save, press [FUNC] + [NO].

    .WORK (Live Working State)

    • Auto-saved constantly as you work
    • Plain text file (you can read/edit it with Python!)
    • Lost if you RELOAD (press [FUNC] + [NO]) or power off without SYNCING

    .STRD (Stored Saved State)

    • Binary file created when you press [FUNC] + [YES] → SAVE
    • Your "checkpoint"—what persists across power cycles
    • Safe and backed up to USB when you sync

    THE GOLDEN RULES

    These rules protect your work. Break them and you'll learn the hard way. Remember: [FUNC] + [YES] is your friend for saving, and always use the OT's own menus (navigate with [LEFT], [RIGHT], [UP], [DOWN] and confirm with [YES]). Check them off:

    • ALWAYS SYNC TO CARD before USB Disk Mode or power off
    • NEVER format CF card from computer — use OT's own menu only
    • NEVER unplug card while playing — press [STOP] first, the OT is streaming from it
    • Audio format: 44.1kHz WAV/AIFF only — 48kHz plays at wrong pitch
    • Keep samples in AUDIO/ folder — OT uses relative paths
    • Use SAVE TO NEW when experimenting — keeps originals safe
    • .work files are not "saved" — press [FUNC] + [YES] to create .strd, which persists across reboots
    • Max 1024 files per folder — organize into subfolders

    Here's your actual CF card structure. You can browse it on the OT by pressing [FUNC] + [AED] to open the Audio Editor, or use [FUNC] + [CUE] to browse the sample pool.

    TEST YOURSELF
    Q1: Where must ALL samples live?
    Correct! The OT uses relative paths (../AUDIO/). Browse it with [FUNC] + [CUE].
    Q2: What's the difference between .work and .strd?
    Correct! .work is volatile, .strd is persistent. Press [FUNC] + [YES] to save .strd.
    Q3: You load a 48kHz sample into the OT. What happens?
    Correct! The OT ONLY works at 44.1kHz. Always convert before loading. Press [FUNC] + [AED] to check sample properties in the Audio Editor.
    Q4: Two patterns in Bank A share the same Part. You edit the machine assignment for Track 1 in Pattern 1. What happens to Pattern 2?
    Correct! Parts are per Bank. Press [PART] to switch Parts and isolate patterns.
