How the Audio Pool Works

Every sample the Octatrack uses must live inside the Set's AUDIO/ folder. When you press [FUNC] + [AED] to open the Audio Editor, or browse samples with [UP] and [DOWN] in the file browser, the OT doesn't reference samples by absolute path—it uses relative paths like ../AUDIO/folder/file.wav .

Critical consequences:

  • Moving samples BREAKS project references. If you move a file, projects that referenced it can't find it.
  • Renaming the Set folder BREAKS everything. The relative path becomes invalid. Never rename the Set folder.
  • Renaming the AUDIO folder BREAKS everything. Same issue.
  • The OT uses FAT32 8.3 short names internally. Long filenames get truncated: !!Samples:Loops becomes !!SAMP~2 .

Best practice: Organize once, move files rarely. If you need to reorganize, do it BEFORE loading samples into projects (before pressing [YES] to assign them to a track), or update project references manually.

Audio Format Requirements

The Octatrack is picky about audio format. If you load a wrong-format file and press [PLAY], it will sound wrong (wrong pitch, wrong speed, or silence). Use ONLY:

  • Format: WAV or AIFF
  • Sample Rate: 44.1 kHz (ONLY—48kHz/96kHz play at wrong pitch/speed)
  • Bit Depth: 16-bit or 24-bit
  • Channels: Mono or Stereo

File Naming Rules

Include BPM for loops: If your filename has a number between 85-170, the OT assumes it's BPM and auto-detects when you open the sample in the Audio Editor with [AED]. Examples:

  • DrumLoop_Dark_128bpm.wav → OT reads 128
  • Arp_Bright_110.wav → OT reads 110
  • Synth_Pad_2024.wav → Warning! OT might think 2024 is BPM (out of range, ignored)

Type Your Filename

Rules:

  • Keep under 30 characters (OT screen truncates longer names)
  • No special chars: colon (:), question mark (?), asterisk (*) — FAT32 limitation
  • Avoid dates in filename if they're in 85–170 range (causes BPM confusion)
  • Use underscores or hyphens instead of spaces (cleaner)

Folder Organization Best Practices

  • Max 1024 files per folder in the OT file browser (navigate with [UP], [DOWN], [LEFT], [RIGHT])—split if needed
  • Organize by type: Drums/, Melodic/, Ambient/, Vocals/
  • Or by BPM: 120bpm/, 130bpm/, 140bpm/
  • Keep path length under ~200 chars (some systems have limits)
  • FAT32 truncates long names: !!Samples:Loops → !!SAMP~2 (watch for collisions)

Flat structure works great too:

AUDIO/
├── Kick_Hard_01.wav
├── Kick_Deep_110.wav
├── Snare_Tight_01.wav
├── Loop_Dark_128bpm.wav
└── [etc]

Pro tip: Use a consistent naming scheme and you won't need deep folders. When you press [YES] to enter a folder in the browser, everything stays fast and visible.

Common Problems & Fixes

Wrong sample rate (48kHz):

Problem: Press [PLAY] and the sample sounds pitched wrong or too slow/fast. Solution: Convert to 44.1 kHz with ffmpeg before loading.

macOS resource forks (._* files):

Harmless but clutter up the file browser when you scroll with [UP]/[DOWN]. Delete with: find PATH -name "._*" -delete

.DS_Store files:

Harmless Finder metadata. Safe to delete. OT ignores them.

Too many files in one folder:

Problem: OT browser limit = 1024 files (you'll see it when browsing with [UP]/[DOWN]). Solution: Split into subfolders.

Path too long:

Problem: Very deep nesting (200+ chars). Solution: Flatten or consolidate.

Projects can't find samples:

Problem: You moved a sample file. Solution: Move back, or re-load the sample by pressing [FUNC] + [AED] to browse again and re-assign it. Check PROJECT > CONTROL > MEMORY for missing references.

Ableton → Octatrack Workflow

Bringing loops from Ableton Live to the OT? Follow this pipeline (you'll use [FUNC], [AED], [PLAYBACK], and [LEVEL] on the OT side):

  1. In Ableton: Warp your loop to the correct tempo.
  2. Consolidate: Right-click loop → Consolidate. This locks in any warping/edits.
  3. Export as exact bars: Export 1, 2, 4, or 8 bar sections (power of 2 helps slicing later).
  4. Format: 44.1 kHz, 16 or 24-bit WAV.
  5. Filename: Include BPM: Loop_Ambient_110bpm.wav
  6. Copy to Set's AUDIO folder: Use Finder or command line. Remember: on the OT, hold [FUNC] and select SYNC TO CARD before entering USB Disk Mode, and press [YES] to confirm.
  7. On OT: Load sample with [FUNC] + [AED], then in the Audio Editor press [AED] again to access Attributes. Set ORIGINAL TEMPO to the BPM.
  8. Timestretch: In Audio Editor Attributes (press [AED] while in the editor), set TIMESTRETCH to NORMAL (tonal) or BEAT (drums).
  9. Playback Setup: Press [PLAYBACK] (or [FUNC] + [AED]) to open playback parameters. Set TSTR to AUTO to enable stretching. Adjust with the [LEVEL] encoder.

Result: Stretch loops to any BPM without losing pitch or quality.
